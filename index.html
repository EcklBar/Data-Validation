1. What is Apache Kafka?

Apache Kafka is a distributed streaming platform based on the Pub-Sub (Publish-Subscribe) model.

Think of it as the Central Nervous System for an organization's data. It decouples data providers (Producers) from data consumers, allowing massive amounts of data to move in real-time.

üí°
Event-Centric Thinking: Instead of processing data in batches at night, Kafka allows us to process events as they occur.

2. Evolution: ETL vs. Streaming

Old World (Traditional ETL)

  • Timing: Batch processing (Nightly/Hourly).
  • Architecture: Monolithic, Point-to-Point.
  • Scalability: Hard and complex.
  • Latency: High (Not Real-Time).

New World (Kafka Streaming)

  • Timing: Real-Time Event Capturing.
  • Architecture: Decoupled, Distributed.
  • Scalability: Fault Tolerant & Parallel Processing.
  • Role: Serves as the "Source of Truth" pipeline.

4. Fundamental Concepts

4.1 Brokers & Cluster

A Broker is a single Kafka server. A Cluster is a group of brokers working together.

  • Connected to one broker? You are connected to the entire cluster.
  • Resiliency: If one broker fails, data is safe on others (due to replication).

4.2 Topics & Partitions

The Topic

A logical category (e.g., "Logs", "Transactions"). Topics are split into Partitions.

The Partition (Crucial!)

Partitions allow parallel processing. Each message gets an incremental ID called an Offset.

‚ö†Ô∏è
Ordering Guarantee: Data order is guaranteed ONLY within a single partition, not across the entire topic.

4.3 Replication

To ensure High Availability (HA), partitions are copied across brokers.

  • Leader: Handles all Reads and Writes for a partition.
  • Follower: Passively replicates data. Takes over if Leader fails.

5. Core APIs

API Name Functionality
Producer API Allows apps to send streams of data to topics.
Consumer API Allows apps to read streams of data.
Streams API Complex transformations (filter, map, aggregate) inside Kafka.
Connect API Reusable connectors for existing systems (e.g., SQL DB to Kafka).

7. Installation Guide (Linux/Mac)

1

Install Java (JDK)

Kafka runs on the JVM. Ensure Java 8+ is installed.

2

Download & Unpack

$ tar -xzf kafka_2.12-2.3.0.tgz
$ cd kafka_2.12-2.3.0
3

Start Zookeeper & Kafka

Run these in separate terminals.

# Terminal 1: Zookeeper
$ bin/zookeeper-server-start.sh config/zookeeper.properties

# Terminal 2: Kafka Server
$ bin/kafka-server-start.sh config/server.properties

8. CLI & Technical Implementation

8.1 Creating a Topic

Use kafka-topics to create a channel for your data.

$ kafka-topics --create \
  --zookeeper localhost:2181 \
  --replication-factor 1 \
  --partitions 1 \
  --topic kafka-test-01

8.2 Real-Time Demo: Producer & Consumer

Open two separate terminal windows to see real-time streaming in action.

Source (Producer)

Type messages here.

$ kafka-console-producer \
--broker-list localhost:9092 \
--topic kafka-test-01

> Hello World
> This is Kafka

Target (Consumer)

Messages appear here instantly.

$ kafka-console-consumer \
--bootstrap-server localhost:9092 \
--topic kafka-test-01

Hello World
This is Kafka
